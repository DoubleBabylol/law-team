## 新增需求

### 需求:省级录入审批路径

系统必须支持一条独立的省级录入审批路径，与现有重大索赔审批路径并行存在。省级录入审批路径为：草稿 → 待省经办审批 → 待省审核 → 已通过。

#### 场景:省级录入事项提交审批

- **当** 索赔事项的审批路径为省级录入，状态为“草稿”或“已驳回”，用户提交审批
- **那么** 系统必须将状态更新为“待省经办审批”
- **那么** 系统必须记录审批日志

### 需求:省经办审批

省经办审批人员必须能够对“待省经办审批”状态的事项进行“通过”或“驳回”操作。

#### 场景:省经办审批通过

- **当** 审批人员对“待省经办审批”的事项点击“通过”
- **那么** 系统必须将状态更新为“待省审核”
- **那么** 系统必须记录审批日志，包含审批人、审批时间、审批意见

#### 场景:省经办审批驳回

- **当** 审批人员对“待省经办审批”的事项点击“驳回”并填写驳回原因
- **那么** 系统必须将状态更新为“已驳回”
- **那么** 系统必须记录审批日志，包含驳回原因

### 需求:省审核

省审核人员必须能够对“待省审核”状态的事项进行“通过”或“驳回”操作。

#### 场景:省审核通过

- **当** 审批人员对“待省审核”的事项点击“通过”
- **那么** 系统必须将状态更新为“已通过”
- **那么** 系统必须记录审批日志

#### 场景:省审核驳回

- **当** 审批人员对“待省审核”的事项点击“驳回”并填写驳回原因
- **那么** 系统必须将状态更新为“已驳回”
- **那么** 系统必须记录审批日志

### 需求:省级审批状态定义

系统必须支持以下新增审批状态：待省经办审批（PENDING_PROVINCE_OFFICE_REVIEW）、待省审核（PENDING_PROVINCE_FINAL_REVIEW）。

#### 场景:状态枚举包含省级状态

- **当** 系统加载审批状态枚举
- **那么** 必须包含 PENDING_PROVINCE_OFFICE_REVIEW（待省经办审批）和 PENDING_PROVINCE_FINAL_REVIEW（待省审核）两个值

### 需求:审批路径类型字段

系统必须在索赔事项中新增审批路径类型字段，用于区分事项走“重大索赔审批”还是“省级录入审批”路径。

#### 场景:新建事项指定审批路径

- **当** 用户创建索赔事项时指定审批路径为“省级录入”
- **那么** 系统必须记录该事项的审批路径为 PROVINCE_ENTRY

#### 场景:已有数据默认路径

- **当** 索赔事项的审批路径字段为空
- **那么** 系统必须将其视为 MAJOR_CLAIM（重大索赔）路径
