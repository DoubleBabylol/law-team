# 为什么

当前系统仅有一条针对重大索赔事项的审批路径（总经办审批 → 总审核）。省级分支机构在录入索赔事项时，缺少省级层面的审核把关环节。需要新增一条独立的"省经办→省审核"录入审批流程，用于省级机构对索赔事项录入的审批管控，与现有的重大索赔审批流程并行存在、互不影响。

## 变更内容

- 新增两个审批状态：`PENDING_PROVINCE_OFFICE_REVIEW`（待省经办审批）、`PENDING_PROVINCE_FINAL_REVIEW`（待省审核）
- 新增独立的省级录入审批流程：草稿 → 待省经办审批 → 待省审核 → 已通过
- 原有重大索赔审批流程保持不变：草稿 → 待总经办审批 → 待总审核 → 已通过
- 省经办和省审核均支持通过和驳回操作
- 需要区分索赔事项走哪条审批路径（省级录入审批 vs 重大索赔审批）

## 功能 (Capabilities)

### 新增功能
- `province-approval`: 省级录入审批流程，包含省经办审批和省审核两个环节，作为独立审批路径与现有重大索赔审批并行

### 修改功能
- `lawsuit-claim-api`: API 需支持新增的省级审批状态，以及根据审批路径类型执行不同的流转逻辑

## 影响

- **后端实体**：`ApprovalStatus` 枚举新增两个省级状态值；`LawsuitClaim` 实体可能需新增审批路径类型字段
- **后端服务**：`LawsuitClaimService.approve()` 方法需根据审批路径类型分别处理流转逻辑
- **后端 API**：审批接口无需新增端点，但内部逻辑需区分审批路径
- **前端**：审批状态展示需适配新状态标签；提交审批时需选择或自动确定审批路径
- **数据库**：已有数据默认为重大索赔审批路径，无需迁移
