## 新增需求

### 需求:表单页面展示

系统必须提供一个"重大索赔事项信息录入"页面，包含"重大索赔事项"和"基本信息"两个模块的所有字段。页面必须使用 Element Plus 表单组件实现。

#### 场景:用户访问填报页面

- **当** 用户导航到重大索赔事项录入页面
- **那么** 系统必须显示包含以下字段的表单：任务名称（文本）、任务编码（只读，自动生成）、任务类型（下拉框）、法院文书号码（文本，50字限制）、是否重大案件（下拉框）、是否理赔给付纠纷（下拉框）、是否退保纠纷（下拉框）、发生区间（下拉框）、被索赔机构省/市/区县/其他（联动选择）、索赔机构（文本）、事实与理由（长文本，1000字限制）、起诉可能性评估（下拉框）、败诉可能性评估（下拉框）、附件（文件上传）

### 需求:任务类型下拉框

任务类型字段必须为下拉框，固定选项为"股份诉讼"、"集团诉讼"、"重大索赔"，禁止用户自行输入其他值。

#### 场景:选择任务类型

- **当** 用户点击任务类型下拉框
- **那么** 系统必须显示且仅显示"股份诉讼"、"集团诉讼"、"重大索赔"三个选项

### 需求:是否类字段下拉框

是否重大案件、是否理赔给付纠纷、是否退保纠纷三个字段必须为下拉框，选项为"是"和"否"。

#### 场景:选择是否类字段

- **当** 用户点击任一是否类字段的下拉框
- **那么** 系统必须显示"是"和"否"两个选项

### 需求:发生区间下拉框

发生区间字段必须为下拉框，显示当季及之前5年的季度选项，格式为"yyyy年N季度"。

#### 场景:查看发生区间选项

- **当** 当前日期为2026年第二季度，用户打开发生区间下拉框
- **那么** 系统必须显示从"2021年一季度"到"2026年二季度"的所有季度选项

### 需求:被索赔机构省市区三级联动

被索赔机构必须支持省、市、区/县三级联动选择。省选项包含全部省及单列市；市选项根据所选省动态加载，包含"无"选项；区/县选项根据所选市动态加载，包含"无"和"其他"选项。

#### 场景:选择省后加载市

- **当** 用户在被索赔机构-省中选择了"广东省"
- **那么** 被索赔机构-市必须显示"无"及广东省下辖所有地级市选项
- **那么** 被索赔机构-区/县必须被清空

#### 场景:选择区县为其他时显示补充输入

- **当** 用户在被索赔机构-区/县中选择了"其他"
- **那么** 系统必须显示"被索赔机构-其他"文本输入框
- **当** 用户在被索赔机构-区/县中选择了非"其他"选项
- **那么** 系统禁止显示"被索赔机构-其他"文本输入框

### 需求:必填字段验证

系统必须对以下字段执行必填验证：任务名称、任务类型、是否重大案件、是否理赔给付纠纷、是否退保纠纷、发生区间、被索赔机构（省/市/区县）、索赔机构、事实与理由、起诉可能性评估、败诉可能性评估。

#### 场景:提交时必填字段为空

- **当** 用户未填写任何必填字段并点击提交
- **那么** 系统必须在每个未填写的必填字段下方显示错误提示信息
- **那么** 系统禁止提交表单

#### 场景:所有必填字段已填写

- **当** 用户填写了所有必填字段并点击提交
- **那么** 系统必须正常提交表单

### 需求:文本字段长度限制

法院文书号码必须限制最大50字，事实与理由必须限制最大1000字。

#### 场景:超出字数限制

- **当** 用户在法院文书号码中输入超过50个字符
- **那么** 系统必须阻止继续输入或显示超出限制的提示

#### 场景:事实与理由超出限制

- **当** 用户在事实与理由中输入超过1000个字符
- **那么** 系统必须阻止继续输入或显示超出限制的提示

### 需求:附件上传

附件字段为非必填项，用户可以上传一个或多个文件作为附件。

#### 场景:上传附件

- **当** 用户点击附件上传区域并选择文件
- **那么** 系统必须将文件上传到服务器并在表单中显示已上传文件列表

#### 场景:删除已上传附件

- **当** 用户点击已上传附件旁的删除按钮
- **那么** 系统必须从列表中移除该附件

### 需求:起诉与败诉可能性评估

起诉可能性评估和败诉可能性评估必须为下拉框，选项为"可能性大"和"可能性小"。

#### 场景:选择可能性评估

- **当** 用户点击起诉可能性评估或败诉可能性评估下拉框
- **那么** 系统必须显示"可能性大"和"可能性小"两个选项
