## 新增需求

### 需求:审批状态管理

每个重大索赔事项必须包含审批状态字段，支持以下状态：草稿（DRAFT）、待总经办审批（PENDING_OFFICE_REVIEW）、待总审核（PENDING_FINAL_REVIEW）、已通过（APPROVED）、已驳回（REJECTED）。

#### 场景:新建索赔事项默认状态

- **当** 用户创建一条新的重大索赔事项
- **那么** 该事项的审批状态必须为"草稿"

### 需求:提交审批

用户必须能够将草稿状态的索赔事项提交审批，提交后状态变为"待总经办审批"。

#### 场景:提交草稿进入审批

- **当** 索赔事项状态为"草稿"，用户点击"提交审批"
- **那么** 系统必须将状态更新为"待总经办审批"
- **那么** 系统必须记录一条审批日志

#### 场景:非草稿状态禁止提交

- **当** 索赔事项状态不为"草稿"或"已驳回"
- **那么** 系统禁止显示"提交审批"按钮

### 需求:总经办审批

总经办审批人员必须能够对"待总经办审批"状态的事项进行"通过"或"驳回"操作。

#### 场景:总经办审批通过

- **当** 审批人员对"待总经办审批"的事项点击"通过"
- **那么** 系统必须将状态更新为"待总审核"
- **那么** 系统必须记录审批日志，包含审批人、审批时间、审批意见

#### 场景:总经办审批驳回

- **当** 审批人员对"待总经办审批"的事项点击"驳回"并填写驳回原因
- **那么** 系统必须将状态更新为"已驳回"
- **那么** 系统必须记录审批日志，包含驳回原因

### 需求:总审核

总审核人员必须能够对"待总审核"状态的事项进行"通过"或"驳回"操作。

#### 场景:总审核通过

- **当** 审批人员对"待总审核"的事项点击"通过"
- **那么** 系统必须将状态更新为"已通过"
- **那么** 系统必须记录审批日志

#### 场景:总审核驳回

- **当** 审批人员对"待总审核"的事项点击"驳回"并填写驳回原因
- **那么** 系统必须将状态更新为"已驳回"
- **那么** 系统必须记录审批日志

### 需求:驳回后重新提交

被驳回的事项必须允许申请人修改后重新提交审批。

#### 场景:重新提交被驳回的事项

- **当** 索赔事项状态为"已驳回"，用户修改内容后点击"重新提交"
- **那么** 系统必须将状态更新为"待总经办审批"
- **那么** 系统必须记录重新提交的审批日志

### 需求:审批日志查看

用户必须能够查看每个索赔事项的完整审批历史记录。

#### 场景:查看审批日志

- **当** 用户点击某条索赔事项的"审批记录"
- **那么** 系统必须显示该事项的所有审批操作记录，包含操作人、操作时间、操作类型、审批意见
