## 一、产品概述

### 1.1 产品背景

随着企业规模扩大和业务流程复杂化，不同岗位、不同部门的人员需要访问不同的系统功能和数据。传统的基于账号的权限管理方式已无法满足精细化管控需求，亟需一套标准化的权限管理系统来实现：

- 角色与权限的灵活配置
- 用户与角色的动态关联
- 菜单与功能的细粒度控制
- 多层级组织架构的支持

### 1.2 产品定位

企业级 RBAC（Role-Based Access Control）权限管理系统，提供完整的权限管理解决方案，支持：

- 多层级组织架构（总公司-分公司-支公司）
- 三级权限粒度（目录-菜单-按钮）
- 多角色组合授权
- 实时权限生效

### 1.3 目标用户

| 用户类型 | 使用场景 | 主要功能 |
|----------|----------|----------|
| 系统管理员 | 系统配置与维护 | 角色管理、用户管理、菜单配置、权限分配 |
| 合规部门人员 | 合规业务处理 | 查看合规相关功能模块 |
| 督察部门人员 | 督察业务处理 | 查看督察相关功能模块 |
| 诉讼部门人员 | 诉讼业务处理 | 查看诉讼相关功能模块 |

### 1.4 业务价值

1. **提升安全性**：通过精细化权限控制，确保用户只能访问授权范围内的功能和数据
2. **降低管理成本**：角色一次配置，多次复用，减少重复授权工作
3. **支持业务扩展**：灵活的角色和权限配置，快速响应组织架构调整
4. **满足合规要求**：完整的权限审计追踪，满足内控合规要求

## 三、功能需求

### 3.1 用户认证模块

#### 3.1.1 登录功能

**功能描述**：
用户通过用户名和密码进行身份验证，验证成功后获取访问令牌（Token），用于后续请求的身份验证。

**业务流程**：
```
用户输入账号密码 → 前端表单校验 → 调用登录接口 → 后端验证
    ↓
验证成功 → 生成 Token → 返回用户信息和 Token → 前端存储 → 跳转工作台
    ↓
验证失败 → 返回错误信息 → 前端展示错误提示
```

**输入要素**：

| 字段 | 类型 | 必填 | 规则说明 |
|------|------|------|----------|
| 用户名 | 文本 | 是 | 已注册的用户名 |
| 密码 | 密码 | 是 | 6-20 位字符 |

**输出结果**：
- 成功：进入系统工作台
- 失败：显示具体错误原因（用户名不存在/密码错误/账号已禁用）

**异常场景**：
1. 账号被禁用：提示"该账号已被禁用，请联系管理员"
2. Token 过期：自动跳转登录页
3. 网络异常：提示网络连接失败

#### 3.1.2 退出登录

**功能描述**：
用户主动退出系统，清除本地登录状态。

**业务流程**：
```
用户点击退出 → 二次确认 → 清除 Token 和用户信息 → 跳转登录页
```

**交互说明**：
- 点击顶栏用户头像下拉菜单中的"退出登录"
- 弹出确认对话框
- 确认后执行退出操作

### 3.2 角色管理模块

#### 3.2.1 角色列表

**功能描述**：
以表格形式展示系统中所有角色，支持查看角色的基本信息和状态。

**列表字段**：

| 字段 | 说明 | 展示方式 |
|------|------|----------|
| 角色 ID | 系统生成的唯一标识 | 文本 |
| 角色名称 | 角色的显示名称 | 文本 |
| 角色类别 | 合规/督察/诉讼 | 带颜色标签 |
| 创建时间 | 角色创建时间 | 日期时间 |
| 状态 | 启用/禁用 | 带颜色标签 |
| 操作 | 编辑、删除 | 按钮组 |

**标签颜色规范**：
- 合规：蓝色
- 督察：橙色
- 诉讼：红色
- 启用：绿色
- 禁用：红色

#### 3.2.2 新增角色

**功能描述**：
创建新的角色，定义角色名称、类别和初始状态。

**输入要素**：

| 字段 | 类型 | 必填 | 规则说明 | 默认值 |
|------|------|------|----------|--------|
| 角色名称 | 文本 | 是 | 2-20 字符，全局唯一 | - |
| 角色类别 | 下拉选择 | 是 | 合规/督察/诉讼 | - |
| 状态 | 开关 | 否 | 启用/禁用 | 启用 |

**业务规则**：
- 角色名称全局唯一，重复时提示错误
- 角色类别创建后允许修改
- 禁用状态的角色不能分配给用户

#### 3.2.3 编辑角色

**功能描述**：
修改已有角色的信息。

**可修改字段**：
- 角色名称
- 角色类别
- 状态

**不可修改字段**：
- 角色 ID
- 创建时间

#### 3.2.4 删除角色

**功能描述**：
删除不再使用的角色。

**业务规则**：
- 已被用户关联的角色不允许删除
- 删除前需二次确认
- 删除角色时同步删除其菜单权限关联

**列表字段**：

| 字段 | 说明 | 展示方式 |
|------|------|----------|
| 菜单名称 | 显示名称 | 文本 + 图标 |

| 操作 | 编辑、添加子项、删除 | 按钮组 |

**类型标签颜色**：
- 目录：蓝色
- 菜单：绿色
- 按钮：橙色

#### 3.4.2 新增目录

**功能描述**：
创建顶级目录，作为菜单树的根节点。

**输入要素**：

| 字段 | 类型 | 必填 | 规则说明 |
|------|------|------|----------|
| 菜单名称 | 文本 | 是 | 2-20 字符 |
| 路径 | 文本 | 否 | 路由路径 |
| 图标 | 图标选择 | 否 | 20 个预设图标 |
| 状态 | 开关 | 否 | 启用/禁用 |

**功能描述**：
在目录或菜单下添加子节点。

**层级规则**：
```
目录
├── 菜单（目录下只能添加菜单）
│   └── 按钮（菜单下只能添加按钮）
└── 目录（同级目录）
```

**目录下添加**：
- 只能添加"菜单"类型
- 类型自动选中且不可修改

**菜单下添加**：
- 只能添加"按钮"类型
- 类型自动选中且不可修改
- 按钮需配置权限标识

#### 3.4.4 编辑菜单

**功能描述**：
修改菜单信息。

**输入要素**：

| 字段 | 类型 | 必填 | 显示条件 |
|------|------|------|----------|
| 菜单名称 | 文本 | 是 | 始终显示 |
| 菜单类型 | 单选 | 是 | 始终显示 |
| 路径 | 文本 | 否 | 类型≠按钮 |
| 权限标识 | 文本 | 否 | 类型=按钮 |
| 图标 | 图标选择 | 否 | 类型=目录或菜单 |
| 状态 | 开关 | 否 | 始终显示 |

**功能描述**：

**业务规则**：
- 删除目录时递归删除所有子菜单和按钮
- 删除前需二次确认
- 删除后影响已分配该菜单的角色权限

| 菜单权限状态 | 全部启用/全部禁用/部分启用 | 标签 |
| 操作 | 配置权限 | 按钮 |

#### 3.5.2 配置菜单权限

**功能描述**：
为角色分配可访问的菜单和按钮权限。

**弹窗内容**：
1. **顶部信息区**：
   - 角色名称、角色组
   - 全部启用/禁用开关
   - 已选择菜单数量

2. **菜单树区域**：
   - 展示完整菜单树（目录-菜单-按钮）
   - 每个节点显示类型标签和图标
   - 支持复选框勾选
   - 父子节点联动选中

**权限继承规则**：
- 选中子节点时，父节点自动半选或全选
- 取消父节点时，所有子节点自动取消
- 保存时记录所有选中的菜单 ID

**全部启用/禁用开关**：
- 全部启用：自动选中所有菜单和按钮
- 全部禁用：取消所有选中
- 操作需二次确认

## 四、业务规则

### 4.1 RBAC 权限模型

系统采用标准的 RBAC（Role-Based Access Control）权限模型：

```
用户 ←→ 角色 ←→ 菜单权限
```

**核心规则**：
1. 一个用户可以拥有多个角色
2. 一个角色可以分配给多个用户
3. 一个角色可以拥有多个菜单权限
4. 用户通过角色间接获得菜单权限

### 4.2 权限粒度

| 粒度级别 | 控制范围 | 示例 |
|----------|----------|------|
| 目录级 | 一级导航菜单的可见性 | 系统管理 |
| 菜单级 | 具体功能页面的访问 | 角色管理页面 |
| 按钮级 | 页面内操作按钮的显示 | 新增角色按钮 |

### 4.3 数据约束

#### 4.3.1 唯一性约束

| 对象 | 唯一字段 | 说明 |
|------|----------|------|
| 用户 | 用户名 | 全局唯一 |
| 用户 | 工号 | 全局唯一 |
| 角色 | 角色名称 | 全局唯一 |

#### 4.3.2 关联约束

| 关联关系 | 约束规则 |
|----------|----------|
| 用户-角色 | 一个用户至少有一个角色 |
| 角色-菜单 | 角色可以没有菜单权限 |
| 角色删除 | 被用户关联的角色不能删除 |
| 菜单删除 | 递归删除所有子菜单 |

### 4.4 状态管理

#### 用户状态

| 状态 | 说明 | 影响 |
|------|------|------|
| 启用 | 正常使用 | 可以登录系统 |
| 禁用 | 暂停使用 | 无法登录，已登录用户被踢出 |

#### 角色状态

| 状态 | 说明 | 影响 |
|------|------|------|
| 启用 | 正常使用 | 可以分配给用户 |
| 禁用 | 暂停使用 | 不能分配给新用户，已分配用户角色失效 |

#### 菜单状态

| 状态 | 说明 | 影响 |
|------|------|------|
| 启用 | 正常显示 | 可以分配给角色 |
| 禁用 | 隐藏显示 | 不能分配给角色 |
